Options

The Ant diff task can take the following options:

NameValueRequired
compare Name of the original file or directory to compare Yes
to Name of the revised file or directory to compare Yes
type Type of the reports to produce, must be one of : normal, rcs or unified No, default to normal
num number of lines of context to output (unified diff only) No, default to 3
outName of a file to which append the reportsNo, default to stdout
errName of a file to which append the errors No, default to stderr or out if out is specified.
recursiveboolean No, used only in the case of folder comparison, when no fileset is specified inside the diff tag.
returnProperty String, name of the property into which the result of the diff will be put, for further use in the build file. No, default to DIFF.
